# Эмпирический анализ MERGE SORT и гибридного MERGE+INSERTION SORT

## Подготовка данных
- Класс `ArrayGenerator` генерирует четыре типовых категории массивов: полностью случайные, заранее отсортированные, отсортированные в обратном порядке и почти отсортированные (5 % случайных перестановок).
- Для каждой категории создаётся опорный массив максимального размера (100 000 элементов). Подмассивы длин 500, 600, …, 100 000 извлекаются как префиксы опорного массива, что гарантирует сопоставимость экспериментов для разных алгоритмов.

## Настройка эксперимента
- В `SortTester` каждый алгоритм сортировки прогоняется один раз на каждой длине и категории данных с использованием `std::chrono::high_resolution_clock`.
- Стандартная реализация `mergeSort` использует дополнительный буфер и полностью рекурсивна.
- Гибридная реализация `hybridMergeSort` переключается на `insertionSort` для подзадач размером не более 15 элементов (значение подобрано из условия задачи A2i).
- Все измерения сохраняются в `results/measurements.csv` в формате CSV для последующей визуализации или статистической обработки.

## Сводные результаты
Средние времена (в миллисекундах) по всем размерам для каждой категории данных:

| Категория          | MERGE SORT | MERGE+INSERTION (t=15) |
|--------------------|-----------:|-----------------------:|
| Случайные          | 2.93       | 2.61                   |
| Отсортированные    | 0.77       | 0.52                   |
| Обратный порядок   | 0.65       | 0.53                   |
| Почти отсортированные | 0.84    | 0.61                   |

Гибридный алгоритм стабильно быстрее стандартной реализации благодаря замене глубоких рекурсий на insertion sort на малых подмассивах. На отсортированных массивах выигрыш особенно заметен из-за высокой локальной упорядоченности данных.

## Повторение эксперимента
1. Соберите консольную утилиту `analysis` (см. инструкции в `README.md`).
2. Запустите полученный исполняемый файл. После завершения работы агрегированная статистика появится в консоли, а подробные измерения будут сохранены в `results/measurements.csv`.
