# Эмпирический анализ MERGE SORT и гибридного MERGE+INSERTION SORT

## Подготовка данных
- Класс `ArrayGenerator` генерирует четыре типовых категории массивов: полностью случайные, заранее отсортированные, отсортированные в обратном порядке и почти отсортированные (5 % случайных перестановок).
- Для каждой категории создаётся опорный массив максимального размера (100 000 элементов). Подмассивы длин 500, 600, …, 100 000 извлекаются как префиксы опорного массива, что гарантирует сопоставимость экспериментов для разных алгоритмов.

## Настройка эксперимента
- В `SortTester` каждый алгоритм сортировки прогоняется один раз на каждой длине и категории данных с использованием `std::chrono::high_resolution_clock`.
- Стандартная реализация `mergeSort` использует дополнительный буфер и полностью рекурсивна.
- Гибридная реализация `hybridMergeSort` переключается на `insertionSort` для подзадач размером не более 30 элементов (значение подобрано из условия задачи A2i).
- Все измерения сохраняются в `results/measurements.csv` в формате CSV для последующей визуализации или статистической обработки.


Гибридный алгоритм стабильно быстрее стандартной реализации благодаря замене глубоких рекурсий на insertion sort на малых подмассивах. На отсортированных массивах выигрыш особенно заметен
